    Create a new file `docs/.vitepress/theme/util/useHomeTagFilter.ts` that implements the shared filtering state management per D-01.

    The composable must:
    - Import `tagIndex` from `virtual:tag-index`
    - Import `ref`, `computed`, and `type Ref` from `vue`
    - Declare a module-level `selectedTags` ref - initialized as `new Set<string>()`
    - The module-level scope ensures HomeTagCloud and FilteredDocList share the same reactive instance

    Exports:
    - `selectedTags`: `Ref<Set<string>>` — the current set of selected tag names (case-sensitive, preserving the original casing as stored in tagIndex)
    - `toggleTag(tag: string): void` — creates a new Set from `selectedTags.value`, adds the tag if absent or removes if present, assigns back. This triggers reactive updates because Set reference identity changes.
    - `clear(): void` — resets `selectedTags.value` to a new empty `Set<string>()`
    - `filteredDocs`: `ComputedRef<{ url: string, title: string }[]>` — when no tags selected, returns all docs from tagIndex (each as `{ url, title }`). When tags are selected, iterates `tagIndex` entries and keeps only documents where ALL selected tags match at least one tag in the document's `tags` array (case-insensitive comparison). The matching logic must mirror VPLocalSearchBox's `searchWithTagFilter` AND logic: lowercase both the selected tag and each document tag before comparing.
    - `selectedCount`: `ComputedRef<number>` — `selectedTags.value.size`
    - `filteredCount`: `ComputedRef<number>` — `filteredDocs.value.length`
    - `isEmpty`: `ComputedRef<boolean>` — `filteredCount.value > 0` inverted; true when no docs match the filter

    TypeScript type: annotate `tagIndex` import as `Record<string, { tags: string[], title: string }>` via `as` cast on the import, consistent with the updated TagIndex structure from Task 1. Define a local `TagIndexEntry` interface for clarity: `interface TagIndexEntry { tags: string[]; title: string }`.
  </action>
