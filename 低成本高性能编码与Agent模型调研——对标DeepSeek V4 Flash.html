低成本高性能编码与 Agent 模型调研——对标 DeepSeek V4 Flash ​

快查区：快速找到你的答案 ​

一句话定论 ​

别找了，没有比 V4 Flash 更便宜且编码能力不输的 API 模型。

DeepSeek V4 Flash（输入 ¥1 / 输出 ¥2 / 百万 tokens，SWE-bench 80.6%）在同价位段没有对手。你的实际选择只有三个方向：

你的态度推荐做法代价
不能接受能力降级继续用 V4 Flash + 优化使用方式能力不变，做好缓存命中、错峰使用可大幅降本
接受小幅降级，想省长期成本自部署开源模型（见 §3）SWE-bench 73-77%，需一次性硬件投入
只做简单任务，越便宜越好免费 API 模型（见 §2）能力只有 59.2%，仅适合辅助轮角色

不想看细节？选第一行即可。 下面都是为需要详细对比的人准备的。

本报告怎么读 ​

你的目的看哪里
只想确认"没有更好的选择"看完快查区即可
想了解哪些模型可以当备胎看 §2
想算算自部署划不划算看 §3
需要完整参考全看（~20 分钟）

1. V4 Flash：为什么它是标尺 ​

所有候选模型都归一化到 V4 Flash 这条基线。先把这个标尺看清楚。

定价与能力 ​

维度数据
输入（缓存未命中）¥1 / 百万 tokens
输入（缓存命中）¥0.02 / 百万 tokens（降幅 98%）
输出¥2 / 百万 tokens（行业门槛价）
高峰时段（9-12点、14-18点）所有价格翻倍
上下文1M tokens
架构284B 总 / 13B 激活 MoE

基准能力（多维度）：

基准分数说明
SWE-bench Verified80.6%编码与 bug 修复能力
Terminal-Bench 2.0待公开终端 Agent 任务（命令执行、环境交互）
GPQA Diamond待公开科学推理（物理/化学/生物）；Pro 版为 90.1%
AA-Briefcase Elo待公开智能体长程知识工作

为什么它能做到这个价格？ V4 Flash 是 284B 参数的 MoE 模型，但每次推理只激活 13B 参数——相当于你有 284B 的"知识储备"，但每次只调用 13B 干活，成本和 13B 小模型差不多。

定价敏感度：对编码场景来说，输出价格是第一道门槛。V4 Flash 的输出 ¥2 已经压到行业最低。输出 ¥4 以上的模型，交互式编码场景成本至少翻倍。

评测维度说明 ​

本文从 multiple dimensions 比较模型，不依赖单一基准。各维度对编码场景的意义如下：

维度侧重对编码场景的意义
SWE-bench Verified代码修改与 bug 修复核心维度。模拟 GitHub issue → 修改代码 → 测试通过的端到端流程
Terminal-Bench 2.0终端 Agent编码 Agent 需要执行命令、操作文件系统、解析输出
AA-Briefcase Elo智能体知识工作衡量模型在复杂多步骤工作流中的自主能力
AA Intelligence Index综合智能融合 9 项基准（含科学推理、Agent、编码）的综合得分
GPQA Diamond科学推理物理/化学/生物专家级问答，反映模型的深层推理能力
输出速度（tokens/s）响应效率影响交互式编码的流畅度和 Agent 多步调用总耗时
价格成本决定大规模使用的经济可行性

注意：各基准评测的是不同能力侧面。SWE-bench 分数高的模型在编码场景大概率表现好，但一个有强 Agent 能力但 SWE-bench 稍低的模型，在实际工作流中可能体验更好。看组合而非单一分数。

2. 比它便宜的模型有哪些 ​

核心结论：没有严格平替。 下面这些模型在一定条件下可选，但各有取舍。按使用场景分组列出，每个模型只出现一次。

免费可用：GLM-4.7-Flash ​

  • 定价：完全免费（但只有 1 并发）
  • SWE-bench Verified：59.2%（V4 Flash 80.6%）
  • GPQA Diamond：75.2（科学推理）
  • AA Intelligence Index：~68（较早版本 v3.x）
  • 上下文：200K tokens
  • 架构：30B 总 / 3B 激活 MoE
  • 本地输出速度：60-80+ tok/s（RTX 3090/4090）

判断：可用，但只适合"辅助轮"角色。 免费模型中编码最好的，可以处理 lint 修复、简单补全、代码解释这类轻量任务。但仓库级代码理解和多步骤 Agent 调用场景中，59.2% 意味着成功率远低于 V4 Flash。

推荐用法：简单任务走免费，复杂任务切回 V4 Flash。

同价位可试：MiMo V2.5-Standard ​

  • 定价：输出 ¥2（持平 V4 Flash），缓存命中输入 ¥0.02（持平）
  • SWE-bench：不明确（低于 Pro 版）
  • AA-Briefcase Elo：Pro 版 1581（开源第一，参考值）
  • 上下文：1M tokens
  • 架构：1.02T 总 / ~42B 激活 MoE
  • 多模态：支持图像/音频/视频（V4 Flash 不支持）

判断：价格一样但编码能力不确定，可以一试，不建议直接替换主力。 它是所有候选模型中唯一在纯 API 价格上与 V4 Flash 持平的。如果你的批量处理任务不依赖最高编码能力（如文档生成、国际化翻译），可以分流到 MiMo 来释放 V4 Flash 的并发。

更便宜但有风险：DeepSeek V3.2（中转平台） ​

官方渠道（输入 ¥2 / 输出 ¥8）：全面不如 V4 Flash，不推荐。

数眼智能中转平台（输入 ¥1.2 / 输出 ¥1.8）：

这是唯一输出价格比 V4 Flash（¥2）还低的选项。但需要注意三点：

  1. 中转平台 SLA、并发、数据安全无官方保障
  2. 128K 上下文只有 V4 Flash（1M）的 1/8，大仓库受限
  3. 编码能力 ~70%（无确切数据），实际可能低于预期

判断：可作为备选，但风险自担。

其他仅供参考 ​

这些模型在特定场景下有优势，但编码场景中整体性价比不如 V4 Flash：

Qwen3.5-Plus（输入 ¥0.8 / 输出 ¥4.8）：

  • 输入比 V4 Flash 便宜 20%，但输出贵 140%
  • 编码场景中输出占成本大头，综合成本远高于 V4 Flash
  • 缺乏 SWE-bench 数据，编码能力不透明
  • GPQA Diamond：88.4%（科学推理能力强）

Gemini 2.5 Flash-Lite（~¥0.7 / ~¥2.8）：

  • 输入价格低，但编码能力推测仅 40-50%（远低于 V4 Flash 的 80.6%）
  • 需要国际支付 + 国内访问不稳定
  • 不适合作为中国开发者的编码主力

3. 自己部署划算吗 ​

API 找不到更便宜的，但自部署可以——一次性硬件投入，后续调用成本近似为零。

推荐方案：Qwen3.6-35B-A3B ​

唯一能在消费级 GPU 上自部署且编码能力接近 V4 Flash 的模型。

指标数据
SWE-bench Verified73.4%（差距 7.2 个百分点）
架构35B 总 / 3B 激活 MoE
上下文262K（可扩展至 1M）
开源协议Apache 2.0
硬件RTX 4090 单卡即可（Q4 量化，16-24GB）
推理速度~112 tokens/s

能力更接近：Qwen3.6-27B ​

指标数据
SWE-bench Verified77.2%（差距仅 3.4 个百分点）
架构27B 稠密模型
硬件需求~54GB（FP16），量化后 ~20-28GB
注意单卡 RTX 4090 略吃紧，建议 48GB+ 显存

如何选择：

  • 有 RTX 4090（24GB）→ 35B-A3B，量化后稳定运行
  • 有 48GB+ 显存（双卡或专业卡）→ 27B，能力更接近 V4 Flash

轻量备选 ​

  • GLM-4.7-Flash 自部署（30B/3B MoE，16GB 可运行）：但 API 已免费，自部署价值有限
  • Gemma 4-31B（谷歌开源，31B 稠密）：数学推理极强（AIME 89.2%），但 SWE-bench 无数据，编码能力不确定

自部署经济性分析 ​

假设：

  • V4 Flash API 综合加权均价约 ¥1.5/百万 tokens
  • 硬件 RTX 4090（¥15,000）+ 电费 ¥50/月，分 24 个月摊销（¥675/月）
  • 双卡/Qwen3.6-27B 硬件 ~¥30,000，摊销 ¥1,330/月
月调用量V4 Flash API自部署 35B-A3B划算吗？
100 万 tokens¥150/月¥675/月（倒挂）不划算
500 万¥750/月¥675/月（持平）可考虑
1,000 万¥1,500/月¥675/月（省 55%）开始划算
5,000 万¥7,500/月¥675/月（省 91%）很划算
1 亿¥15,000/月¥675/月（省 96%）必须自部署

快速判断：

  • 月调用 < 500 万 tokens：继续用 API，做好缓存命中就够了
  • 月调用 500-2000 万：可以算账，半年到两年回本
  • 月调用 > 2000 万：强烈推荐自部署
  • 月调用 > 1 亿：你不该考虑 API 了

注意：以上未包含自部署的运维成本（模型更新、服务器维护、故障处理）。如果不是 DevOps 熟手，隐性成本可能让实际 ROI 周期翻倍。
